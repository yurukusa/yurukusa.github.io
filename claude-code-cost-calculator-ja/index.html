Claude Code の月額の費用の計算の道具

月の Anthropic の請求の見立てを出す。 機械的な作業を安い副の作業者に流した場合 (Path B の併用) の節約の見立ても出す。 留まるか、 併用するか、 乗り換えるかを、 数字で判定する。

最終更新 2026-05-05。 値段は 2026-04-22 以後の Anthropic の Pro 版の構造の変更と、 現在の API の単価で計算する。 併用の節約の比は 2026-05-05 の r/ClaudeAI の 81 票 の検証 (217 件 の機械的な作業、 DeepSeek V4 Flash で 0.41 ドル、 Sonnet で約 7 ドル、 17 分の 1 の比) を用いる。

1. 今のプランは何か。
現在の Anthropic の購読に合うものを選ぶ。 4 月 22 日 以後の値段を使う。
2. 1 日 のうち、 Claude Code を実際に動かす時間はどれくらいか。
作業のセッションが活発な時間。 端末を開いたまま放置の時間は含めない。
6 時間 / 日
3. 機械的な作業の割合はどれくらいか。
整形、 分類、 要約、 抽出。 安い模型でできるが、 今は Sonnet に流している作業の割合。

この数字は見立てで、 実測ではない。 実際の費用は、 トークンの混ざり方、 cache の当たり方、 模型の振り分け、 1 セッションごとの cache_creation の量で変わる。 Token Book の第 1 章 が、 自分の実際の支出の測り方を扱う。 ここの見立ては 800 時間 を超える Claude Code の自律の運用の median の値で計算している。 併用の節約は、 副の作業者を正しく設定した場合の数字 (CLAUDE.md の deny 形のルール、 出力の人による確認) を前提にする。 確認なしで本番に流すのは危険。
