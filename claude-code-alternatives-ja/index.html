ホーム 3 冊の本 痛みの一覧 English

2026 年 5 月 の Claude Code の代替の選択肢。
感想ではなく、 数字で見る比較。

5 つの道具が Claude Code の作業の領域を共有している。 値段、 公開の頻度、 向く利用者は、 それぞれ違う。 この頁は 5 つを並べて、 一つの問いに正直に答えるための材料を出す。 留まるか、 併用するか、 乗り換えるか。

最終更新 2026-05-05。 出典: GitHub の公開の API、 各社の値段の頁、 r/ClaudeAI の直近 24 時間、 Hacker News の直近 7 日間の上位。

早見の比較
道具 公開の頻度 値段の形 表面 向く利用者
Claude Code 毎日 (2026-05-05 で v2.1.123、 alpha はもっと頻繁) 月 20 から 200 ドル、 加えて週次の上限あり CLI と各種の編集器の拡張 計画と実行を 1 つの道具で行いたい、 質に対して払う意欲のある利用者
Cursor 毎日 (閉源、 GitHub の公開の頁なし) 月 20 ドル、 Pro Plus は 40 ドル VS Code の派生、 CLI なし 編集器の中で完結したい、 GUI の操作の方が安心な利用者
Cline 毎週 (2026-04-24 で v3.81.0、 直近 30 日 で 5 件 公開) BYOK、 自分の模型の業者に直接払う VS Code の拡張 Claude Code の自律の感じが好きで、 模型の支払いを自分で管理したい利用者
Codex CLI 非常に活発 (rust v0.125.0 安定、 alpha は毎日) OpenAI の Plus か Pro の購読に含まれる、 加えて API CLI 既に OpenAI の利用者で、 Claude Code と同等の道具を同じ shell で使いたい利用者
Aider 停滞 (2025-08-09 で v0.86.0、 9 ヶ月 公開なし) BYOK、 自分の模型の業者に直接払う CLI 既に安定の作業の流れがある利用者、 新しい機能は不要な利用者
DeepSeek の wrapper 新しい (DeepClaude が 2026-05-04 から 24 時間で 941 stars) DeepSeek のトークン、 同等の作業で Sonnet の約 17 分の 1 の費用 Claude Code の呼び出しを差し替える wrapper Claude Code の shell を残したまま、 後端を安く差し替えたい利用者。 Sonnet 級の品質で良い利用者

DeepSeek は wrapper を使わなくても直接利用できる。 Claude Code の環境変数 ANTHROPIC_BASE_URL に api.deepseek.com/anthropic を、 ANTHROPIC_AUTH_TOKEN に DeepSeek の鍵を入れる。 公式の DeepSeek の文書にこの手順が書いてある。 wrapper の方は経路の整理と切り替えの便利さが追加される。 多くの場合は環境変数の経路の方が単純。

道具ごとの判定

Cursor

閉源 ・ 編集器の中 ・ 月 20 から 40 ドル

これは何か

VS Code の派生で、 Anthropic の模型を Cursor の自前の後端に通す。 売りは編集器との一体感。 自分の作業の場所に AI が居る。

Claude Code から乗り換える理由

  • VS Code を毎日使っているなら、 移行の摩擦がほぼ無い
  • エージェントの面が編集器の中で動く、 編集器を離れない
  • 値段が決まっている、 Claude Max より安い

Claude Code に戻る理由

  • script 化できない、 CLI が無い
  • 大きなリポでの複数ファイルの再構成は Claude Code の CLI のエージェントの方が速い場合がある
  • 閉源で、 何がいつ送られるかを監査できない
判定: 編集器の中で完結する作業の流れと、 1 つの値段で済ませたい場合は強い候補。 端末からの自律の作業や CI からの呼び出しが中心の場合は弱い候補。

Cline

VS Code の拡張 ・ BYOK ・ 2026-04-24 で v3.81.0

これは何か

VS Code の拡張で、 Claude Code に近いエージェントの動きをするが、 模型の鍵は自分で持つ。 Anthropic、 OpenAI、 DeepSeek、 OpenRouter、 ローカルの模型を、 同じ画面で切り替えられる。

Claude Code から乗り換える理由

  • 模型の請求が、 中継の業者ではなく直接の業者の請求書で見える、 上乗せが無い
  • 同じセッションの中で、 作業ごとに別の模型に振り分けられる
  • 毎週の公開の頻度、 直近 30 日 で 5 件 の公開

Claude Code に戻る理由

  • BYOK は、 全ての枠の問題を自分で扱う必要がある
  • VS Code の中で動く、 端末からの背景の実行はできない
  • エージェントの動きは良いが、 Claude Code の自動の処理よりは少し手動寄り
判定: Claude Code に近い自律の感じを、 Anthropic の請求の枠の外で得たい場合は強い候補。 端末で見守りなしの実行が必要な場合は弱い候補。

Codex CLI

CLI ・ OpenAI ・ rust v0.125.0 安定、 alpha は毎日

これは何か

OpenAI の CLI のエージェント。 2026 年 に Rust で書き直された。 Claude Code と同じ役割を担う。 端末で長時間のセッション、 ファイルの編集、 道具の呼び出し、 計画と実行。

Claude Code から乗り換える理由

  • 既に OpenAI の購読が中心なら、 請求が 1 本にまとまる
  • 4 月 の Rust 化で、 alpha の公開が毎日続いている。 勢いが本物
  • 2026 年 を通して、 周辺の道具の生態系が拡大している

Claude Code に戻る理由

  • エージェントの個性が違う。 CLAUDE.md を整えていた人は、 o 系の模型に対して別の整え方が必要
  • Sonnet が得意な作業 (長文の生成、 多数のファイルの不変条件を保つ編集など) は、 別の指示の出し方が必要
  • 毎日の alpha の公開は、 退行の頻度も上がる。 本番では安定の版に固定する方が良い
判定: 既に OpenAI の購読者で、 同じ shell の中に Claude Code 相当の道具を欲しい場合は強い候補。 Sonnet 向けの指示の整え方に投資した場合は弱い候補。

Aider

CLI ・ BYOK ・ 2025-08-09 で v0.86.0 (9 ヶ月 停滞)

これは何か

CLI の作業の道具の元祖。 Claude Code より数年早く存在した。 軽量で意見の強い設計、 BYOK。

現在の状態

最後の公開は 2025 年 8 月 9 日 の v0.86.0。 2026 年 5 月 の時点で、 9 ヶ月 公開が無い。 主要な開発者の GitHub の活動も別の方向に移った。 報告の入口は受け付けているが、 取り込みは止まっている。 Aider は安定の状態だが凍結の状態と見るのが良い。

今でも合う場合

  • Aider の特定の編集の形に依存した作業の流れがあって、 それが動いている場合
  • 新しい機能が不要で、 自分の下で変わらない道具が欲しい場合
  • 必要なら自分で fork を維持する余裕がある場合
判定: 既に動いている古い作業の流れには合う。 2026 年 5 月 に新しく採用するなら、 Cline や Codex CLI を選ぶ方が良い。

DeepSeek (直接 / wrapper)

環境変数 か wrapper ・ DeepSeek V4 ・ Sonnet の同等の作業で約 17 分の 1 の費用

これは何か

2 つの経路がここで合流する。 ひとつは Claude Code の環境変数 ANTHROPIC_BASE_URL と ANTHROPIC_AUTH_TOKEN を、 DeepSeek の Anthropic 互換の口に向ける経路。 もうひとつは aattaran/deepclaude のような wrapper (2026-05-04 公開、 24 時間で 941 stars) を経由する経路。 どちらも結果として、 Claude Code の CLI の見た目はそのままで、 後端だけが安い模型に変わる。

乗り換える理由

  • 3 週間 で Sonnet なら約 7 ドル かかる作業の量が、 DeepSeek V4 Flash なら約 0.41 ドル。 17 分の 1 の比は実測で再現できる
  • Claude Code の見た目は変わらない、 CLAUDE.md を書き直す必要が無い
  • 週次の上限は関係なくなる、 DeepSeek にトークン単位で直接払う

戻る理由

  • 品質は Sonnet 級。 Opus が必要な作業はこの経路では満たせない
  • 次の Anthropic の CLI の公開で、 環境変数の優先の順番や API の形が変わって、 経路が壊れる可能性がある。 これは仮の話ではなく現実の懸念
  • Anthropic 固有の口に依存する機能 (一部の MCP の道具、 特定の道具の呼び出しの形) が使えなくなる
判定: Sonnet 級の品質で十分な大量の機械的な作業で、 請求が問題の場合は強い候補。 Opus の推論や Anthropic の機能の全集に依存する場合は弱い候補。
留まるか / 併用するか / 乗り換えるか を決める 5 つの問い

留まるか、 併用するか、 乗り換えるかは、 感想ではなく数字で答える問い。 数字を出すための 5 つの問いはこれ。

直近 30 日の 1 つのマージ済の作業の単位あたりの費用はいくらか。 答えられないなら、 Token Book の第 1 章 が測定の手順を扱っている。 この数字が無いと、 残りの問いの答えは全部、 推測になる。
Claude Code の利用のうち、 どれくらいの割合が、 安い模型でできる機械的な作業か。 40 パーセント を超えるなら、 併用の経路 (機械的な作業を DeepSeek に、 判断の作業を Claude に振り分ける) が最大の効果。 r/ClaudeAI の 2026-05-05 の 81 票 の投稿が測定の手順を示した。
本番の出力の不確かさへの自分のチームの耐性はどれくらいか。 低いなら、 ここで挙げた全ての道具が苛立たせる。 解決は道具ではなく仕組み。 hook、 事後の確認、 黄金経路の検査。 Postmortems 本 が 12 件 の事故事例で、 まさにこの隙間が痛みになった事例を扱う。
自分の作業のうち、 Sonnet 級ではなく Opus 級の推論が必要な作業はどれくらいか。 多いなら、 Claude Code の Max の購読と Cursor の Pro Plus だけが、 安定して Opus を提供する。 DeepSeek と Cline の経路は、 今の段階では Sonnet 級が上限。
乗り換えにかかる人の手の時間は、 どれくらいか。 CLAUDE.md の整え直し、 hook の入れ直し、 MCP の設定の置き換え、 チームの教育。 ここを正直に見積もってから決める。 Migration Playbook の第 3 章 にこの見積りの表がある。
この頁の背景にある 2 件の出来事

2026 年 4 月 22 日。 Anthropic が Pro 版の構造を変えて、 週次の上限の入れ方が変わった。 重い利用者が今までと同じ感覚で使うと、 上限に当たる場所が変わる。 Hacker News の話 (id 47963204) は 1,338 ポイント を超えて伸び続けている。 反応は「Claude Code が悪い」 ではなく「自分のチームの計算が合わなくなった」 だった。

2026 年 5 月 4 日。 aattaran/deepclaude が GitHub に作られた。 24 時間で 941 stars。 r/ClaudeAI の話は 626 ポイント、 265 件 のコメント。 これが動くのは、 Claude Code の CLI の見た目が良いから、 後端だけ別の模型に差し替えても使い続けたいから。

この 2 つの組み合わせが、 2026 年 5 月 を分岐の月にした。 CLI は愛されている、 計算は合わない、 代替の道具が動く状態にある。 Migration Playbook の 5 月 8 日 の無料更新は、 この頁の 3 日後。

3 冊の本、 自分の経路に合わせて

この判定が難しいのは、 1 つの判定ではないから。 少なくとも 3 つの判定がある。 何にいくら払うか、 何を動かすか、 何かが壊れた時にどうするか。 この 3 つの層を、 3 冊の本がそれぞれ扱っている。

Migration Playbook: 4 つの経路を並べて、 それぞれの利点と難点を、 業者の宣伝の頁では見られない形で扱う。 2026 年 4 月 25 日 から発売中、 5 月 8 日 に既存の購入者向けの無料更新を配信。 117 ページ、 19 ドル。

Token Book: 自分の経路の費用の見積もりと、 実際の支出の測定の方法。 94 ページ、 自分の判断で 5 ドル 以上 の値段。

Incident Postmortems: 12 件 の本物の事故を、 検出と防止の hook つきで再構成した本。 100 ページ の PDF。 値段は商品の頁を見てほしい。

3 冊を一緒に見たい場合は、 Operations Suite の頁 が 3 種類 の読者の経路 (費用の調査、 乗り換えの判定、 事故の検視) を整理している。 各本は Gumroad で個別に売っている、 まとめ売りの値引きはない、 整理の頁は道案内のためのもの。
