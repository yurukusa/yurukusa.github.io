Nine-Axis Verification Tracker

Self-audit tool for Claude Code's claim-vs-reality divergence pattern. Snapshot: 2026-05-18.

What this is. Nine independent verification axes that document the structural pattern where Claude Code's response surface reports success while the underlying runtime diverges from the claim. Five axes are inside the vendor (Anthropic's own publications, leaks, and changelog). Four axes are outside (CVE registrations, security media, top community signals, alternative-tool emergence).

How to use it. Read each axis card. If you have personally observed the pattern that axis documents — in your own Claude Code use, in a project you maintain, in a system you are reviewing — check the box on the card. The summary panel at the bottom counts your observations and lets you export a personal verification record (plain text, copy-paste, never sent anywhere). All state stays in your browser's local storage.

Why nine axes. One report is anecdote. A few reports can be coincidence. Nine independent channels — five internal to the vendor, four external — make the pattern structural rather than incidental. The point is not to count failures. The point is to show that when nine separate verification paths converge on the same conclusion, the conclusion has earned the right to be treated as a structural property of the current system.

Why this tool exists

    
For most of the past six weeks, operators have reported individual incidents — a regression here, a permission bypass there, an unexpected billing event over the weekend. Each report is filed under its own bug number. Each gets its own fix in a subsequent release. The structural pattern across the incidents — that the tool's response surface and the runtime have diverged in ways that the operator cannot detect without external verification — has not had an equally durable name.


    
This tracker assigns the pattern a name (claim-vs-reality divergence) and lists nine independent verification axes for the pattern's existence. If you find an additional axis, the source is listed in each card's reference link — fork the underlying Gist and add yours.
