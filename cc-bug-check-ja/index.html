2026年5月19日の朝までの48時間で、anthropics/claude-codeの起票の場に5件の異なる不具合が報告された。5つの質問に答えると、自分の運用がどの不具合に遭遇している可能性があるかと、それぞれの対処の経路が分かる。

この頁はブラウザの中で全件を実行し、何もネットワークに送信しない。ページの読み込みの段の標準のGitHub Pagesの記録の他に、追跡の道具は無い。

5つの質問

1. あなたのClaude Codeの版は2.1.143以降ですか?

claude --versionで確認できる。findとgrepの入れ子の作業者の起動の不具合 (起票#60325) は2.1.143で初めて再現された。

2. ~/.claude/CLAUDE.md またはプロジェクトのCLAUDE.mdに「Xはしない」という明示の規則を書いていますか?

例: 「填し言葉(『あなたは正しい』 等)を出すな」 「rm -rfは絶対に使うな」 「確認なしにmainへpushするな」。 規則が文脈に読み込まれている状態で、模型がセッションの中で規則を違反した経験がある場合、起票#60339の事例に該当する。

3. CLAUDE.mdでTaskCreate / TaskListの使用を明示で禁じていますか?

別の作業の追跡の仕組みを使っているために、 CLAUDE.mdで「Do NOT use TaskCreate」 のような明示の禁止を書いている場合に該当。 仕組みの側が定期的に「TaskCreateの使用を考慮しろ」 の促しを差し込み、 CLAUDE.mdの opt-out を素通りする経路の起票#60323に遭遇する。

4. CLAUDE.mdに「セッションの開始の段で必ずXを実行する」 のような手順を書いていますか?

例: 「セッションの開始の段で特定のファイルを作成」 「文脈の読み込みを実行」。 規則が読み込まれた状態で、 手順が沈黙で省略される(エラーも警告もなし)経験がある場合、 起票#60337に該当する。 模型は手順が完了したかのように作業の段を進める。

5. Claude Codeを安全の検証の作業(CVE再現、 exploitの起草、 侵入試験)で使っていますか?

命令の正確性が決定的に重要な作業。 模型がセッションの中で確立した正しい命令 (例: ./chage_pwn root) を参照せず、 別の蓋然性の高い命令 (例: ./exploit) を本文に出力した経験がある場合、 起票#60340に該当する。 再現の手順の引き渡しの文書に特に危険。

長い整理が必要な場合

この5問の診断は、2026年5月18-19日に報告された5件の不具合を対象にする。 出典の本であるClaude Code Claim-Verify Handbookでは、 GitHubの起票の場の130件の事例、 14件の防衛の手順、 5件の自動の点検の道具を整理する。 2026年5月22日発売、 19米ドル、 約89頁のPDF。
